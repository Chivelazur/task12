Требования к формату файлов с результатами:

1) Файлы с результатами должны размещаться в каталоге Results, в котором должны быть три каталога: Ground, Drop, Camera.

2) В каталоге Ground должны быть размещены файлы с расписанием работы наземных станций.
Каждое имя файла - Ground_<название станции>.txt. Например, Ground_Anadyr1.txt
--- export_sessions

3) В каталоге Drop должны быть размещены файлы с расписанием сброса данных для каждого спутника.
Каждое имя файла - Drop_<имя спутника>.txt. Например, Drop_KinoSat_110101.txt

4) В каталоге Camera должны быть размещены файлы с расписанием работы камеры спутника.
Каждое имя файла - Camera_<имя спутника>.txt. Например, Camera_KinoSat_110101.txt

5) Формат файлов Ground_<название станции>.txt:
Anadyr1 - имя станции
-------------------------
Access * Start Time (UTCG) * Stop Time (UTCG) * Duration (sec) * Sat name * Data (Mbytes)
1 1 Jun 2027 00:00:01.000 1 Jun 2027 00:04:21.296 260.296 KinoSat_110101 100
2 …

5) Формат файлов Drop_<имя спутника>.txt:
KinoSat_110101 - имя спутника
-------------------------
Access * Start Time (UTCG) * Stop Time (UTCG) * Duration (sec) * Station * Data (Mbytes)
1 1 Jun 2027 00:00:01.000 1 Jun 2027 00:04:21.296 260.296 Anadyr1 100
2 …

5) Формат файлов Camera_<имя спутника>.txt:
KinoSat_110101 - имя спутника
-------------------------
Access * Start Time (UTCG) * Stop Time (UTCG) * Duration (sec) * Data (Mbytes)
1 1 Jun 2027 00:00:01.000 1 Jun 2027 00:04:21.296 260.296 100
2 …

6) Общие замечания:
 - все расширения файлов .txt
 - в названия спутников "KinoSat_" перед номером - обязательно
 - объем переданных/заполненных данных - в Мегабайтах !!!
 - разделители - пробелы (можно несколько), допускается символ табуляции
 - строка дефисов может содержать любое кол-во дефисов (но ничего кроме дефисов)
 - количество секунд (Duration) должно соответствовать времени начала и конца
